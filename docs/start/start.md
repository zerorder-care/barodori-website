# 바로도리 홈페이지 PRD
## 1. 목적

영아 사경을 의심하는 보호자에게 신뢰할 수 있는 정보를 제공한다.

앱 설치 수와 서비스의 인지도, 신뢰도를 높인다.

## 2. 배경

인스타그램, 네이버 블로그, 지인 추천, 검색 등을 통해 바로도리를 알게 된 보호자가 자세한 정보를 확인할 수 있는 공식 채널이 필요하다.

오가닉 유입을 늘리고 어플의 인지도를 높인다.

## 3. 주요 사용자

### 3.1 1차 사용자

인스타그램, 블로그, 병원 / 지인 추천 등을 통해 바로도리를 알게 된 후 서비스 신뢰도를 확인하려는 보호자

### 3.2 2차 사용자

영아 사경이 의심되는 0~12개월 영아의 보호자

### 3.3 페르소나

강지현, 31세 여성

생후 3개월 아기를 키우고 있으며, 아이가 한쪽으로만 고개를 돌리는 것 같아 영아 사경을 검색 중이다.

네이버 블로그에서 바로도리를 알게 되었고, 앱 설치 전 신뢰할 수 있는 서비스인지 확인하기 위해 홈페이지에 방문한다.

## 4. 핵심 사용자 여정

### 4.1 홈페이지 직접 유입

홈 → 영아 사경 설명 → 바로도리 제품 소개 → 안전 안내 → 앱 설치

### 4.2 아티클 검색 유입

검색 결과 → 사경 아티클 → 관련 아티클 탐색 → 바로도리 소개 → 앱 설치

## 5. 핵심 페이지

- 홈
- 제품 소개
- 사경 아티클 목록 / 상세
- 앱 설치 / 베타 (서포터즈) 신청 페이지
- 개인정보처리방침 / 이용약관
- 데스크탑: QR로 설치 화면 넘어가는 모달

## 6. 주요 지표

- 오가닉 검색 유입 수
- 아티클 조회수
- 앱 설치 CTA 클릭 비율 + 실제 설치율
- 평균 체류 시간
- 검색 노출 수
- 검색 클릭 비율
- 인스타/블로그 유입 대비 앱 설치 클릭 비율

## 7. 기능 요구사항

- 홈페이지에서 바로도리 서비스의 핵심 기능을 이해할 수 있어야 한다.
- 영아 사경의 대표 증상과 병원 진료가 필요한 상황을 확인할 수 있어야 한다.
- 아티클을 통해 사경, 두상 비대칭, 가정 운동, 월령별 관리 정보를 탐색할 수 있어야 한다.
- 모든 주요 페이지에서 앱 설치로 이동할 수 있어야 한다.
- 신규 아티클을 추가하거나 기존 콘텐츠를 수정할 수 있어야 한다.
- 아티클은 SEO에 적합한 고유 URL과 메타 정보를 가져야 한다.

## 8. 비기능 요구사항

- 모바일 우선 반응형 웹
- LCP 2초 이하
- INP 200ms 이하
- CLS 0.1 이하
- Lighthouse SEO 95점 이상
- sitemap.xml 제공
- robots.txt 제공
- OG 이미지 및 메타 태그 제공
- 네이버 서치어드바이저 및 구글 서치콘솔 등록 가능
- 동시접속자 1,000명 수준의 트래픽을 안정적으로 처리

## 9. 의료/안전 고지

- 바로도리는 의료기관의 진단을 대체하지 않는다.
- 통증, 강한 울음, 발열, 호흡 이상, 신경학적 이상, 급격한 움직임 제한 등이 있는 경우 즉시 병원 진료를 권장한다.
- 보호자가 임의로 강한 힘을 가해 스트레칭하지 않도록 안내한다.
- 운동 콘텐츠는 전문가 검토를 거친 범위 내에서 제공한다.

## 10. 분석 요구사항

- 앱 설치 CTA 클릭 이벤트
- 앱 설치 CTA 클릭 → 실제 앱 설치 추적
- 아티클 조회 이벤트
- 아티클 → 제품 소개 이동 이벤트
- 스크롤 깊이 이벤트
- 외부 채널별 UTM 추적
